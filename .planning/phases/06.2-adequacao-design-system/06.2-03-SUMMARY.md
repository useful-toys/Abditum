- `createVaultSaveBeforeMsg` type added (not reusing an existing message) to keep semantics clear and avoid coupling with other flows.
- `stateSaveAsNew` in `saveAndExitFlow` is the cleanest way to guard `filePickerResult` without introducing flow coupling — the flow only handles picker results when explicitly in that state.
- `mockManagerForInit` (test helper) not wired to a real `*vault.Manager` — tests for the dirty-check branch (desvio 6) are marked skip with explanation because `newCreateVaultFlow` takes `*vault.Manager` directly and cannot accept a mock interface.
