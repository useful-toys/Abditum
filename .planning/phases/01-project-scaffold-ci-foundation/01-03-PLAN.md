Create `.golangci.yml` with required linters and security rules.

**Linters to enable** (per RESEARCH §6):
- errcheck (unchecked errors)
- govet (standard checks)
- staticcheck (static analysis)
- revive (style + best practices)
- gosec (security issues)
- gocritic (performance + style)
- forbidigo (forbidden patterns)

**Forbidden patterns** (per arquitetura.md §5):
- `math/rand` → MUST use `crypto/rand` for security
- `net` → network calls prohibited
- `net/http` → HTTP calls prohibited

**Configuration for stub packages:** Since packages currently only have doc.go files with no code, configure to avoid false positives on empty packages.

```yaml
linters:
  enable:
    - errcheck
    - govet
    - staticcheck
    - revive
    - gosec
    - gocritic
    - forbidigo

linters-settings:
  forbidigo:
    forbid:
      - p: 'math/rand'
        msg: 'Use crypto/rand instead of math/rand for security-critical randomness'
      - p: '^net\.'
        msg: 'Network calls are prohibited - application must be offline-only'
      - p: '^net/http\.'
        msg: 'HTTP calls are prohibited - application must be offline-only'
  
  errcheck:
    check-blank: true
  
  govet:
    enable-all: true
  
  staticcheck:
    checks: ["all"]
  
  revive:
    rules:
      - name: exported
        disabled: false
      - name: package-comments
        disabled: false

issues:
  exclude-rules:
    # Allow doc.go files to have minimal content
    - path: doc\.go
      linters:
        - revive
      text: "package comment"
    
    # Empty packages are intentional stubs in Phase 1
    - path: internal/
      linters:
        - gocritic
      text: "commentFormatting"

run:
  timeout: 5m
  go: '1.26'
```

**Why forbid math/rand:** Per arquitetura.md §5, all randomness must use crypto/rand for security (salts, nonces, IDs).

**Why forbid net/net.http:** Per arquitetura.md §5, application is offline-only by design — network calls are security violations.

**Why exclude-rules for doc.go:** Phase 1 stubs are intentionally minimal. Linters may complain about package comment format — we exclude those specific warnings for doc.go files to avoid false positives.
