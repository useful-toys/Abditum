  <action>
Create `.github/workflows/ci.yml` with 3 jobs: build, lint, test.

**CRITICAL:** Set `CGO_ENABLED: 0` as environment variable at job level (NOT just in command) — per RESEARCH §4 and §8.2, this ensures `go test` also uses static linking.

Triggers: `push` and `pull_request` to `main` branch (per CONTEXT D-03).

Go version: `1.26.x` in setup-go action (per CONTEXT D-02).

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  build:
    name: Build
    runs-on: ubuntu-latest
    env:
      CGO_ENABLED: 0
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
        with:
          go-version: '1.26.x'
      - name: Build binary
        run: go build -v ./cmd/abditum
      - name: Verify static linking
        run: |
          file ./abditum
          ldd ./abditum 2>&1 | grep -q "not a dynamic executable" || (echo "ERROR: Binary is dynamically linked" && exit 1)

  lint:
    name: Lint
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
        with:
          go-version: '1.26.x'
      - name: golangci-lint
        uses: golangci/golangci-lint-action@v6
        with:
          version: latest

  test:
    name: Test
    runs-on: ubuntu-latest
    env:
      CGO_ENABLED: 0
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
        with:
          go-version: '1.26.x'
      - name: Run tests
        run: go test ./... -race -count=1 -v
```

**Why `CGO_ENABLED: 0` at job level:** Ensures the environment variable applies to ALL commands in the job, not just build. This is critical — without it, `go test` might succeed with dynamic linking, hiding portability issues (per RESEARCH §8.2 pitfall).

**Why verify static linking:** The build job includes explicit verification that the binary is statically linked (`ldd` check) — catches if CGO_ENABLED somehow didn't apply.

  <action>
Create `Makefile` with standard development targets: build, test, lint, vet, clean.

**build target:** Uses `CGO_ENABLED=0`, respects `GOOS` from environment, outputs binary named `abditum`.

**test target:** Includes `-race` flag for race detection, `-count=1` to disable test caching.

**lint target:** Runs golangci-lint.

**vet target:** Runs go vet.

**clean target:** Removes build artifacts.

```makefile
.PHONY: build test lint vet clean

BINARY=abditum
GOOS?=$(shell go env GOOS)
GOARCH?=$(shell go env GOARCH)

build:
	CGO_ENABLED=0 GOOS=$(GOOS) GOARCH=$(GOARCH) go build -o $(BINARY) ./cmd/abditum

test:
	CGO_ENABLED=0 go test ./... -race -count=1 -v

lint:
	golangci-lint run

vet:
	go vet ./...

clean:
	rm -f $(BINARY)
```

**Why `GOOS?=$(shell go env GOOS)`:** Allows override via environment (`make GOOS=windows build`) while defaulting to current OS.

**Why `-count=1` in test:** Disables Go's test cache, ensuring tests always run fresh.

**Why `CGO_ENABLED=0` in both build and test:** Enforces static linking consistently (per RESEARCH §5 and arquitetura.md §5).
