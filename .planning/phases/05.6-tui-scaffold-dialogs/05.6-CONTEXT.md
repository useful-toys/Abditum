## Current State

### Compilation Errors (decision.go)

**Error 1:** `decision.go:179:2: declared and not used: titleStyle`

The `titleStyle` variable is declared but never used — the title is embedded in
`BorderTitle()` call which doesn't exist.

**Error 2:** `decision.go:206:3: borderStyle.BorderTitle undefined`

`lipgloss.Style` in lipgloss v2 has no `BorderTitle` method. The canonical way to embed
a title in the top border is to **manually construct** the top border line:

```
╭── ⚠  Excluir segredo ───────────╮
```

This line must be hand-built: `╭──` + styled_title + fill_dashes + `╮`

### Fix Strategy

The `View()` method must:

1. **Build top border manually** — `╭── symbol title ────...────╮`
2. **Build middle lines** — `│  ` + body_line + trailing_spaces + `  │`
3. **Build blank padding lines** — `│  ` + spaces + `  │` (1 line top, 1 line bottom)
4. **Build action bar** — already implemented correctly in `renderActionBar()`
5. **Assemble** as `strings.Join(lines, "\n")`

The bottom border is already handled by `renderActionBar()` which returns the complete
`╰── ...actions... ──╯` line.

### Word Wrap Strategy

Body text must be wrapped to `boxWidth - 4` columns (border 2 + padding 2).
Use a simple word-wrap loop (no external deps). Preserve explicit `\n` in the body.

## Decisions

### D-01: Manual border construction (no lipgloss BorderTitle)

lipgloss v2 has no `BorderTitle` method. Top border is built character by character:
```
╭── {symbol} {title} ──...──╮
```
Where `──...──` is filled with `─` to reach `boxWidth` exactly.

The left anchor is always `╭── ` (4 chars: corner + 2 dashes + space).
The right anchor is always ` ──╮` (4 chars: space + 2 dashes + corner).
If `symbol != ""` the title is `symbol + "  " + title` (double space between symbol and text).
Total fill = `boxWidth - leftAnchorW - titleW - rightAnchorW`.
If fill < 1, use 1 (never negative).

### D-02: Body wrapping

`wrapBody(text string, maxWidth int) []string` — splits on spaces and existing `\n`.
Returns `[]string` of lines, each ≤ `maxWidth` runes.
No external dependency — stdlib only.

### D-03: Action bar is already correct

The existing `renderActionBar()` implementation in `decision.go` is correct and should not
be changed (it's already building the bottom border manually).

### D-04: Test assertions on View() output

Tests must NOT rely on ANSI escape codes to verify token colors (fragile, platform-dependent).
Instead, assert on **plain-text content** — symbol presence, title text, action labels, border chars.
Token color tests are deferred to Phase 06+ visual integration.
