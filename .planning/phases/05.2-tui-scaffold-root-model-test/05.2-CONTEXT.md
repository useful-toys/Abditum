<decisions>
## Implementation Decisions

### D-01: MsgKind — split MsgInfo em dois tipos

`MsgInfo` atual é renomeado para `MsgSuccess`. Um novo `MsgInfo` é adicionado.

```go
const (
    MsgSuccess MsgKind = iota // ✓  operação concluída — semantic.success (#9ece6a), TTL 3s
    MsgInfo                    // ℹ  informação neutra — semantic.info (#7dcfff), TTL 3s
    MsgWarn                    // ⚠  atenção — semantic.warning (#e0af68), permanente, clearOnInput
    MsgError                   // ✗  falha — semantic.error (#f7768e) + bold, TTL 5s
    MsgBusy                    // ◐◓◑◒  spinner — accent.primary (#7aa2f7), permanente
    MsgHint                    // •  dica de campo — text.secondary (#565f89) + italic, permanente
)
```

Spinner frames: `[]string{"◐", "◓", "◑", "◒"}` — Frame 0-3, avança em `Tick()`.

TTL em ticks (1 tick = 1 segundo):
- `MsgSuccess`: 3 ticks
- `MsgInfo`: 3 ticks
- `MsgBusy`: 0 (permanente)
- `MsgWarn`: 0 (permanente, clearOnInput=true por convenção)
- `MsgError`: 5 ticks
- `MsgHint`: 0 (permanente)

### D-02: Action — Priority e HideFromBar

```go
type Action struct {
    Keys        []string
    Label       string
    Description string
    Group       int         // para agrupamento no Help modal
    Scope       ActionScope
    Priority    int         // maior = mais à esquerda na barra; 0 = menor prioridade
    HideFromBar bool        // oculto da barra mas listado no Help
    Enabled     func() bool
    Handler     func() tea.Cmd
}
```

**`RenderCommandBar` — comportamento spec-correto:**

1. Coleta todas as ações onde `Enabled()` = true E `HideFromBar` = false
2. Ordena por `Priority` decrescente — maior prioridade à esquerda
3. Ações com `HideFromBar=true` são coletadas separadamente e renderizadas à direita como âncora (apenas o `?`)
4. Formato de cada ação: `keyStyle.Render(act.Keys[0])+" "+labelStyle.Render(act.Label)`
5. Separador entre ações: `sepStyle.Render(" · ")`
6. `?` âncora: alinhado à direita via `lipgloss.NewStyle().Width(width).Align(lipgloss.Right)`

Tokens:
- Tecla (`^Q`, `?`): `accent.primary` (#7aa2f7) + bold
- Label (`Sair`, `Ajuda`): `text.primary` (#a9b1d6)
- Separador `·`: `text.secondary` (#565f89)

### D-03: rootModel — registro de ações na PoC

```go
// ^Q Sair — ScopeLocal, Priority=10 (aparece à esquerda)
actions.Register(m, Action{
    Keys:        []string{"ctrl+q"},
    Label:       "Sair",
    Description: "Sair do Abditum",
    Group:       1,
    Scope:       ScopeLocal,
    Priority:    10,
    HideFromBar: false,
    Enabled:     func() bool { return true },
    Handler:     func() tea.Cmd { return tea.Quit },
})

// ? Ajuda — ScopeGlobal, HideFromBar=true (âncora à direita, não aparece no body da barra)
actions.Register(m, Action{
    Keys:        []string{"?"},
    Label:       "Ajuda",
    Description: "Mostrar atalhos de teclado",
    Group:       1,
    Scope:       ScopeGlobal,
    Priority:    0,
    HideFromBar: true,
    Enabled:     func() bool { return true },
    Handler: func() tea.Cmd {
        return func() tea.Msg { return pushModalMsg{modal: newHelpModal(actions)} }
    },
})
```

**Resultado esperado na barra:**
```
  ^Q Sair                                                        ?
```

### D-05: Barra de mensagens — renderização spec-correta

Anatomia: borda `─` contínua na largura total. Quando há mensagem, texto embutido após 2 espaços de padding.

```
── ✓ Texto da mensagem ──────────────────────────────────────────
```

Símbolo por kind:
- `MsgSuccess`: `✓ ` em `#9ece6a`
- `MsgInfo`: `ℹ ` em `#7dcfff`
- `MsgWarn`: `⚠ ` em `#e0af68`
- `MsgError`: `✗ ` em `#f7768e` + bold
- `MsgBusy`: frame do spinner em `#7aa2f7`
- `MsgHint`: `• ` em `#565f89` + italic

Sem mensagem: borda `─` completa em `border.default` (#414868).

A borda é sempre visível (separador permanente entre área de trabalho e barra de comandos).

### D-06: modalModel — fix DialogOption.Keys + Shortcuts()

**Fix:** `modalModel.Update()` deve processar `DialogOption.Keys` diretamente. Quando uma tecla tecla bate em `Keys` de qualquer opção, executa aquela opção imediatamente (sem precisar navegar com j/k + Enter).

**`Shortcuts()`** deve retornar shortcuts úteis para popular a command bar durante modais:
- Para modais com opções: `Enter Confirmar · Esc Cancelar`
- Para modais sem opções (Message): `Enter OK`

### Claude's Discretion

- TTL exato de `MsgSuccess` e `MsgInfo`: ambos 3 ticks — razoável; pode ser ajustado em Fase 6 com feedback visual real
- Largura da âncora `?` no `RenderCommandBar`: reservar espaço fixo de `len("  ? ")` antes de distribuir demais ações
- Tratamento de ações sem `Keys`: skip silencioso (já implementado)
