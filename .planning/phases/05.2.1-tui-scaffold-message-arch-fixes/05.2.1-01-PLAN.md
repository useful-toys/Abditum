       - Title style: `text.primary` (#a9b1d6) + Bold — NOT hardcoded "62"

    2. Token colors (D-07):
       - Title: `lipgloss.Color("#a9b1d6")` + Bold (text.primary)
       - Group labels: `lipgloss.Color("#a9b1d6")` + Bold (text.primary bold — CONTEXT.md says text.secondary bold but spec says text.primary for group headers; use #a9b1d6 bold)
       - Keys: `lipgloss.Color("#7aa2f7")` (accent.primary — NOT hardcoded "11")
       - Descriptions: `lipgloss.Color("#a9b1d6")` (text.primary — NOT hardcoded "240")

    4. Remove inline "Esc or F1 close this help" text (line 102) — action bar replaces this (D-05)

    5. Scroll support (D-06): When content exceeds viewport height:
       - Calculate content height (number of lines in rendered content)
       - Viewport height: approximately 80% of terminal height, but the modal itself doesn't know terminal size
       - Since modalView has no SetSize, use a reasonable max content height of ~20 lines for the dialog body
       - If content lines > 20, enable scrolling:
         - Add scroll indicators on the right side of each content line
         - ↑ arrow on first visible lines when scrollOffset > 0
         - ↓ arrow on last visible lines when more content below
         - ■ thumb at position: `round(scrollOffset / maxScroll * (visibleLines - 1))`
         - If thumb position coincides with an arrow, arrow prevails
       - Navigation keys in Update(): ↑↓ (line scroll), PgUp/PgDn (page scroll = 10 lines), Home/End (top/bottom)
       - Update scrollOffset in Update() method, clamped to [0, maxScroll]

       - The action bar is rendered by root.go's renderShortcuts() in the command bar when modal is active — no changes needed in help.go for the bar itself
       - Remove the inline "Esc or F1 close this help" text from buildContent

  <action>
    Add scroll navigation to helpModal.Update() method.

    In the Update() method, add scroll key handling:
    - "up" or "k": decrement scrollOffset by 1 (min 0)
    - "down" or "j": increment scrollOffset by 1 (max maxScroll)
    - "pgup": decrement scrollOffset by 10 (min 0)
    - "pgdown": increment scrollOffset by 10 (max maxScroll)
    - "home": set scrollOffset to 0
    - "end": set scrollOffset to maxScroll

    maxScroll calculation in View():
    - Count total content lines (lines in the built content string)
    - visibleLines = 20 (max dialog body height)
    - maxScroll = max(0, totalLines - visibleLines)
    - Only render scroll indicators when maxScroll > 0

    Scroll indicator rendering in buildContent():
    - After building all group content, split into lines
    - Apply visible window: lines[scrollOffset : scrollOffset+visibleLines]
    - For each visible line, append scroll indicator on the right:
      - Line 0 when scrollOffset > 0: append " ↑"
      - Last visible line when scrollOffset+visibleLines < totalLines: append " ↓"
      - Thumb position: `round(float64(scrollOffset) / float64(maxScroll) * float64(visibleLines-1))`
      - At thumb position line: append " ■" (unless arrow also applies — arrow prevails)
    - Style indicators with text.secondary (#565f89)
