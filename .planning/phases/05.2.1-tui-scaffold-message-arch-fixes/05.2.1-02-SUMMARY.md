### Task 2: Command Bar Truncation Logic
- Rewrote `RenderCommandBar()` body building logic in `actions.go` with priority-based truncation:
  - F1 anchor always stays visible ‚Äî space reserved before distributing body actions
  - Body actions truncated by priority when they don't fit in available width
  - Separator " ¬∑ " width included in fit calculation
  - If even highest-priority action + F1 don't fit ‚Üí show only F1
  - Recalculates on every render (no caching)
- Implementation: builds body parts one at a time, tracking cumulative width, stops when next part + separator would exceed `width - anchorW - 1`

### Task 3: PoC Binary Modal Pattern
- Replaced `showHelp bool` toggle with `modals []pocModal` stack in `pocMessagesModel`
- Added `pocModal` local interface matching tui package's `modalView` contract
- F1 handler now pushes/pops modals instead of toggling bool
- Update() dispatches to top modal when stack is active, checks for `isModalResult()` to pop
- View() uses `lipgloss.Place` to center modal dialog when stack is active
- Command bar shows modal shortcuts (via `renderPocShortcuts`) when modal active, otherwise shows `RenderCommandBar()`
- Exported `NewHelpModal()` constructor in help.go enables PoC to instantiate help modal
- Removed unused `fmt` import
