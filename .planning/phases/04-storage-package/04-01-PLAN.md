  <action>
    Per D-02 from CONTEXT.md: Add two new functions to `internal/crypto/aead.go`.

    **`EncryptWithAAD(key, plaintext, aad []byte) (nonce []byte, ciphertext []byte, err error)`:**
    - Validate key is 32 bytes → `ErrInvalidParams`
    - Create AES cipher block, create GCM wrapper
    - Generate fresh 12-byte nonce via `io.ReadFull(rand.Reader, nonce)` → `ErrInsufficientEntropy` on failure
    - Call `gcm.Seal(nil, nonce, plaintext, aad)` — note: pass `nil` as dst (not nonce like current Encrypt does), because nonce is returned separately
    - Return `(nonce, sealedCiphertext, nil)`

    **`DecryptWithAAD(key, ciphertext, nonce, aad []byte) ([]byte, error)`:**
    - Validate key is 32 bytes → `ErrInvalidParams`
    - Create AES cipher block, create GCM wrapper
    - Validate nonce length matches `gcm.NonceSize()` → `ErrAuthFailed`
    - Call `gcm.Open(nil, nonce, ciphertext, aad)` — on error return `ErrAuthFailed`
    - Return plaintext

    Key difference from existing Encrypt/Decrypt: nonce is NOT prepended to ciphertext output. Nonce and ciphertext are separate return values because the storage layer writes them to different positions in the binary file (nonce in header bytes 37-48, ciphertext as payload after header).

    Write TDD-style: tests first (RED), then implementation (GREEN). Follow existing test patterns in aead_test.go.
