## Phase Boundary

Fix residual bugs discovered after Phase 05.2/05.2.1:

### Message Bar (3 bugs)
1. **root.go não usa RenderMessageBar()** — `renderFrame()` renderiza texto cru sem símbolo, sem borda contínua, sem cores semânticas

6. **Sem barra de ação na borda inferior** — spec pede "Esc Fechar" na borda inferior do diálogo, não só na command bar

### Espaçamento e Cor da Mensagem (2 bugs)
7. **Espaçamento entre ícone e texto** — spec: exatamente 1 espaço entre símbolo e início do texto
8. **Cor do texto da mensagem** — spec: "Token se aplica à mensagem inteira — símbolo e texto usam o mesmo token de cor"

<decisions>
## Implementation Decisions

### Bug 1: root.go deve usar RenderMessageBar()
- **D-01:** Substituir `msgBarStyle.Render("  " + msgText)` em `root.go:314` por chamada a `tui.RenderMessageBar(m.messages.Current(), m.width)`
- **D-02:** Remover `msgBarStyle` dedicado — `RenderMessageBar()` já cuida de toda a estilização (borda, símbolo, texto, fill)
- **D-03:** Manter `msgBarH = 1` — a barra continua com 1 linha de altura

### Bug 2: Texto deve usar mesmo token semântico do símbolo
- **D-04:** Aplicar cor semântica ao texto inteiro da mensagem — símbolo E texto usam o mesmo token (spec: "Token se aplica à mensagem inteira — símbolo e texto usam o mesmo token de cor")
- **D-05:** Cada MsgKind mantém sua cor para símbolo + texto:
  - `MsgSuccess`: `#9ece6a` (`semantic.success`)
  - `MsgInfo`: `#7dcfff` (`semantic.info`)
  - `MsgWarn`: `#e0af68` (`semantic.warning`)
  - `MsgError`: `#f7768e` + Bold (`semantic.error`)
  - `MsgBusy`: `#7aa2f7` (`accent.primary`)
  - `MsgHint`: `#565f89` + Italic (`text.secondary`)

### Bug 3: Truncamento de mensagens longas
- **D-06:** Truncar texto com `…` (U+2026) quando excede largura disponível
- **D-07:** Usar `lipgloss.Width()` para cálculo de largura visual — NUNCA `len(s)` (anti-padrão)
- **D-08:** Borda `─` deve permanecer contínua e fechar corretamente em qualquer largura
- **D-09:** Truncar no final do texto, antes do suffix da borda
- **D-10:** Mínimo útil: se width < prefix + symbol + space + suffix, mostrar só símbolo + borda (sem texto)

### Espaçamento entre símbolo e texto
- **D-11:** Exatamente 1 espaço entre símbolo e início do texto (spec: "exatamente 1 espaço entre o símbolo e o início do texto")
- **D-12:** Padding esquerdo da barra: 2 espaços antes do símbolo (align com texto do cabeçalho)

### Bug 4: Help modal — F1 empilha modal em vez de fechar
- **D-13:** F1 com modal aberto deve fechar, não empilhar outro — a ação F1 tem `ScopeGlobal` e é interceptada pelo `ActionManager.Dispatch` antes do modal receber input
- **D-14:** Solução: quando modal está ativo, F1 deve ser ignorado pelo Dispatch OU o modal deve interceptar F1 antes do Dispatch

### Bug 5: Help modal — ESC não fecha
- **D-15:** Verificar dispatch order — modal deve receber input quando está no topo da pilha
- **D-16:** Se o modal não está recebendo input, corrigir o fluxo em `root.go:187-188`

### Bug 6: Help modal — sem barra de ação na borda inferior
- **D-17:** Adicionar "Esc Fechar" na borda inferior do diálogo (spec: `╰── Esc Fechar ──╯`)
- **D-18:** A barra inferior é parte do diálogo, não da command bar — renderizada dentro do `View()` do helpModal

### Teste de truncação com F10
- **D-19:** Criar ação F10 no PoC (`cmd/poc-mensagens/poc_model.go`) que exibe `MsgInfo` com texto > 100 caracteres
- **D-20:** A mensagem longa deve validar visualmente o truncamento com `…` (U+2026) quando excede largura da barra
- **D-21:** Texto de teste sugerido: "Esta é uma mensagem de informação muito longa que excede o limite de 100 caracteres e deve ser truncada corretamente na barra de mensagens sem corromper a borda."

### the agent's Discretion
- Exato algoritmo de truncamento (char-by-char vs binary search)
- Se adicionar testes unitários para truncamento em `messages_test.go`
- Comportamento em terminais muito estreitos (< 20 colunas)

</decisions>
