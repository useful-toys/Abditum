  <action>
    In root.go Update(), the dispatch order currently runs ActionManager.Dispatch BEFORE checking modals. This means F1 (ScopeGlobal, Priority 0) always fires and pushes another help modal on top of the existing one.

    Fix: Check if the topmost modal is a helpModal BEFORE running ActionManager.Dispatch for the F1 key. When a helpModal is active, F1 should NOT trigger the action — let the modal handle it instead.

    Change the dispatch order in the `tea.KeyPressMsg` case (around line 176):

    CURRENT:
    ```go
    // 1. ActionManager.Dispatch
    if cmd := m.actions.Dispatch(key, inFlowOrModal); cmd != nil {
        return m, cmd
    }
    // 2. Topmost modal
    if len(m.modals) > 0 {
        return m, m.modals[len(m.modals)-1].Update(msg)
    }
    ```

    REPLACE WITH:
    ```go
    // 1. If help modal is open, let it handle ALL keys (including F1 and ESC)
    if len(m.modals) > 0 {
        if _, isHelp := m.modals[len(m.modals)-1].(*helpModal); isHelp {
            return m, m.modals[len(m.modals)-1].Update(msg)
        }
    }
    // 2. ActionManager.Dispatch
    if cmd := m.actions.Dispatch(key, inFlowOrModal); cmd != nil {
        return m, cmd
    }
    // 3. Other modals
    if len(m.modals) > 0 {
        return m, m.modals[len(m.modals)-1].Update(msg)
    }
    ```

    This ensures:
    - When helpModal is on top, it receives ALL key input (F1 toggles close, ESC closes)
    - When helpModal is NOT on top, F1 triggers the action (pushes new help modal)
    - Other modals continue to work as before

    Do NOT modify help.go Update() — it already handles "esc" and "f1" correctly by returning popModalMsg. The bug is purely in root.go dispatch order.

    In help.go View(), add a bottom action bar to the dialog per spec (D-17, D-18).

    The spec requires: `╰── Esc Fechar ──╯` as the bottom border of the dialog — this is PART of the dialog, not the command bar.

    Current View() returns:
    ```go
    return lipgloss.NewStyle().
        Border(lipgloss.RoundedBorder()).
        BorderForeground(lipgloss.Color("62")).
        Padding(0, 1).
        Width(boxW).
        Render(content)
    ```

    Modify to include the action bar. The approach:
    1. Build the main content (title + shortcuts + separator) as before
    2. Create an action bar line: `Esc Fechar` centered with border dashes
    3. Stack content + action bar vertically
    4. Apply border to the combined content

    Implementation:
    ```go
    func (m *helpModal) View() string {
        boxW := 60

        allActions := m.actions.All()
        content := m.buildContent(allActions)

        // Bottom action bar (D-17): "Esc Fechar" with border fill
        barStyle := lipgloss.NewStyle().Foreground(lipgloss.Color("#565f89"))
        actionText := barStyle.Render("Esc Fechar")
        // Calculate fill: boxW minus padding (2) minus border chars (2) minus action text width
        borderWidth := boxW - 4 // account for border + padding
        actionWidth := lipgloss.Width(actionText)
        fillEach := (borderWidth - actionWidth) / 2
        if fillEach < 1 { fillEach = 1 }
        fill := strings.Repeat("─", fillEach)
        actionBar := barStyle.Render(fill + " ") + actionText + barStyle.Render(" " + fill)

        combined := content + "\n" + actionBar

        return lipgloss.NewStyle().
            Border(lipgloss.RoundedBorder()).
            BorderForeground(lipgloss.Color("62")).
            Padding(0, 1).
            Width(boxW).
            Render(combined)
    }
    ```
