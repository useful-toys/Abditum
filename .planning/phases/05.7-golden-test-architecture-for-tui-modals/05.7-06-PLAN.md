<objective>
Fix the ANSI parser to produce output conforming to the spec in arquitetura-teste.md.

Three bugs exist:
1. `styleMapToArray` returns nil (not `[]string{}`) when no styles are active — JSON emits `null` instead of `[]`
2. `StyleTransition` marshals as a verbose object `{"line":0, "col":0, ...}` instead of the compact tuple `[0, 0, fg, bg, [styles]]`
3. Duplicate transitions can appear at the same `(line, col)` position (parser emits a transition before counting the text chars, then the next SGR at the same position overwrites it without removing the first)

Purpose: The parser is the foundation. All downstream golden files depend on it producing the correct format. Until the format is correct, no golden file can be considered spec-compliant.

Output: Fixed `ansiparser.go` with custom JSON marshaling and dedup logic.
</objective>

Add a `MarshalJSON() ([]byte, error)` method to `StyleTransition` that emits the compact 5-element array format required by the spec:

```go
// MarshalJSON serializes StyleTransition as a compact 5-element tuple:
//   [line, col, fg_hex_or_null, bg_hex_or_null, [styles]]
// This matches the spec in arquitetura-teste.md § "Formato de cada tupla".
func (st StyleTransition) MarshalJSON() ([]byte, error) {
    type tuple [5]any
    var fg, bg any
    if st.FG != nil { fg = *st.FG }
    if st.BG != nil { bg = *st.BG }
    return json.Marshal(tuple{st.Line, st.Col, fg, bg, st.Style})
}
```

**Fix 1 — Empty style array (not nil):**

In `styleMapToArray`, replace the `var active []string` declaration with:
```go
active := []string{}
```
This ensures that when no styles are active, the return value is an empty (non-nil) slice. With `MarshalJSON` emitting `st.Style` directly via `json.Marshal`, this guarantees the JSON output is `[]` not `null`.

**Fix 2 — Deduplicate transitions at same (line, col):**

The current parser records a transition whenever `newStateKey != lastStateKey` — but this check happens BEFORE advancing `col` past the text. If two SGR sequences appear at the same column position (col hasn't moved between them), the second transition overrides the first in `result` without removing the previous entry at that same position.

Replace the transition-recording block in `ParseANSIStyle`:

```go
// Current (buggy):
if newStateKey != lastStateKey {
    transition := StyleTransition{...}
    result = append(result, transition)
    lastStateKey = newStateKey
}

// Fixed: also deduplicate same (line,col) by replacing last entry if at same position
if newStateKey != lastStateKey {
    transition := StyleTransition{
        Line:  line,
        Col:   col,
        FG:    currentState.fg,
        BG:    currentState.bg,
        Style: styleMapToArray(currentState.style),
    }
    // If the last recorded transition is at the same (line, col), replace it
    // (multiple SGR codes at the same position → only the final state matters).
    if len(result) > 0 && result[len(result)-1].Line == line && result[len(result)-1].Col == col {
        result[len(result)-1] = transition
    } else {
        result = append(result, transition)
    }
    lastStateKey = newStateKey
}
```

Also fix the `MarshalJSON` to handle nil Style safely — when `st.Style` is nil (from old entries), output `[]string{}`:
```go
style := st.Style
if style == nil { style = []string{} }
return json.Marshal(tuple{st.Line, st.Col, fg, bg, style})
```
