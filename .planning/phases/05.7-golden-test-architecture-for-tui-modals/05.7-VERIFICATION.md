## Notable: `commandbar-no-actions-*.json.golden` Contains `null`

Two `.json.golden` files contain the JSON literal `null` rather than `[]`:

- `commandbar-no-actions-30.json.golden` â†’ `null`
- `commandbar-no-actions-60.json.golden` â†’ `null`

**This is expected and acceptable.** When `RenderCommandBar([]Action{}, w)` produces an empty string (no ANSI at all), `ParseANSIStyle("")` returns a nil `[]StyleTransition` slice (the `var result []StyleTransition` declaration). `json.MarshalIndent(nil, ...)` encodes a nil slice as JSON `null`.

The arquitetura-teste.md spec does not explicitly address the zero-transition case. The SUMMARY.md for plan 08 documents this as a known and acceptable anomaly: *"JSON null is valid for no-action commandbar scenarios (nil slice â†’ JSON null)"*. The tests pass for these files (byte-for-byte stable on second run).

**No action required** â€” the `no-actions` scenario represents a degenerate edge case with genuinely no content, and `null` is semantically accurate (no transitions exist).
