# Phase 05.7: Golden Test Architecture - Plan

## Goal

Implement golden test architecture for 4 TUI components with 36 pairs (72 files) of golden tests, parser SGR, 2 refactorings, and 16 Update() test scenarios.

## Tasks

### T1: Parser SGR (`internal/tui/testdata/ansiparser.go`)
- Parse ANSI escape sequences from `View()` output
- Track state: fg, bg, bold, italic, underline, strikethrough, faint, blink, reverse
- Normalize 256-color and truecolor to hex
- Record transitions ONLY when visual state changes (skip redundant/identical states)
- Return `[]StyleTransition` — each with line, col, fg, bg, style array
- Unit tests for parser itself

### T2: Refactor `RenderCommandBar` to pure function
- Change signature: `func (a *ActionManager) RenderCommandBar(width int) string` → `func RenderCommandBar(actions []Action, width int) string`
- Update call site in `root.go`: `m.actions.RenderCommandBar(m.width)` → `RenderCommandBar(m.actions.Visible(), m.width)`
- Update existing tests in `actions_test.go` (3 tests)
- Verify `go test ./internal/tui/...` passes

### T3: Refactor `helpModal` to decouple from ActionManager
- Change constructor: `newHelpModal(actions *ActionManager)` → `newHelpModal(actions []Action, groupLabel func(int) string)`
- Update `View()`: `m.actions.All()` → use `m.actions` field directly (now `[]Action`)
- Update `buildContentLines()`: receive `[]Action` directly
- Update `totalLines()`: use `m.actions` directly
- Update `GroupLabel` calls: use `m.groupLabel(grp)` function
- Update call site in `root.go`: `newHelpModal(m.actions.All(), m.actions.GroupLabel)`
- Verify `go test ./internal/tui/...` passes

### T4: Golden test for MessageBar
- Create `TestRenderMessageBar_Golden` in `messages_test.go`
- 6 kinds × 2 sizes = 12 test cases
- Fixtures with ~45-char messages per kind
- Generate 12 `.txt.golden` + 12 `.json.golden` files
- Run `-update` to generate, verify visually

### T5: Golden test for CommandBar
- Create `TestRenderCommandBar_Golden` in `actions_test.go`
- 5 scenarios × 2 sizes = 10 test cases
- Fixtures: `noActions`, `f1Only`, `typicalActions`, `unsortedActions`, `manyActions`
- Generate 10 `.txt.golden` + 10 `.json.golden` files
- Run `-update` to generate, verify visually

### T6: Golden test for HelpDialog
- Create `TestHelpDialog_ViewGolden` in `help_test.go` (new file)
- 8 scenarios:
  - 3 actions, no scroll: 30 cols, 60 cols
  - 15 actions, scroll: top(0%), mid(40%), bottom(100%) × 30 cols, 60 cols
- Fixtures: `help3actions()`, `help15actions()`, `groupLabel()`
- Generate 8 `.txt.golden` + 8 `.json.golden` files
- Run `-update` to generate, verify visually (especially bottom scroll — last line visible)

### T7: Golden test for DecisionDialog
- Create `TestDecisionDialog_ViewGolden` in `decision_test.go`
- 10 combined scenarios (not cartesian)
- Use existing `pocKey1`-`pocKeyF` fixtures + new ones for long titles/2-line bodies
- Generate 10 `.txt.golden` + 10 `.json.golden` files
- Run `-update` to generate, verify visually

### T8: Update() tests for DecisionDialog
- Add `TestDecisionDialog_Update` in `decision_test.go`
- 8 scenarios: Enter(1/2/3-action), Esc(1/2-action), explicit key, unknown key
- Verify cmd return values (non-nil for action, nil for unknown)

### T9: Update() tests for HelpDialog
- Add `TestHelpDialog_Update` in `help_test.go`
- 8 scenarios: Esc, F1, Up(clamp), Down(clamp), PgDown, Home, End
- Verify scroll state changes and popModal return

### T10: Verify all tests pass
- `go build ./...`
- `go test ./internal/tui/... -v` — all existing + new tests pass
- `go test ./internal/tui/... -update` — regeneration works without errors
- Commit all changes
