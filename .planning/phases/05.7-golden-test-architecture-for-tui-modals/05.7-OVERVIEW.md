# Fase 05.7: Arquitetura de Testes Golden para Modais TUI

## Objetivo da Fase

Implementar uma arquitetura de testes **golden files** para validar rendering visual de componentes modais (layout, espaçamento, cores, fontes), detectando regressões visuais que testes convencionais não capturam.

**Aplicar:** DecisionDialog (24 pares) + HelpDialog (8 pares iniciais) = 64 golden files.

## Contexto

- **Problema:** Bug de espaçamento em `DecisionDialog.renderActionBar()` passou em testes existentes (commit `6325967`)
- **Raiz:** Testes só verificam `strings.Contains()` — não capturam espaçamento, alinhamento, cores
- **Solução:** Dois arquivos golden por caso de teste:
  - `.txt.golden` — layout visual (byte-a-byte)
  - `.json.golden` — transições de estilo (agnóstico a SGR)

## Deliverables

1. **Parser SGR reutilizável** (`internal/tui/testdata/ansiparser.go`)
   - Extrai transições visuais de output ANSI
   - Normaliza cores 256-color e truecolor para hex
   - Registra apenas mudanças reais (não redundâncias)

2. **Golden test runner para DecisionDialog**
   - Matriz: 5 severidades × 3 layouts × 2 tamanhos = 24 pares
   - 48 arquivos golden em `internal/tui/testdata/golden/decision-*.{txt,json}.golden`
   - Flag `-update` para regeneração

3. **Golden test runner para HelpDialog**
   - Matriz: 2 variantes × 2 tamanhos (inicial) = 4 pares
   - 8 arquivos golden em `internal/tui/testdata/golden/help-*.{txt,json}.golden`
   - Extensível para mais variantes futuras

4. **Documentação**
   - `arquitetura-teste.md` (já criado) — especificação genérica
   - `05.7-PLAN.md` — plano de implementação
   - `05.7-UAT.md` — testes de aceitação

## Critério de Sucesso

✅ Parser SGR compila e extrai transições corretamente  
✅ DecisionDialog: 44 testes existentes passam + 1 novo golden test (24 sub-testes)  
✅ HelpDialog: 8 testes existentes passam + 1 novo golden test (4 sub-testes)  
✅ Todos os 48 golden files (DecisionDialog) + 8 (HelpDialog) gerados e commitados  
✅ `go test ./internal/tui/... -update` regen funciona sem erros  
✅ `go test ./internal/tui/...` passa 100%  

## Escopo Futuro

- Estender golden tests a novos modais conforme forem criados
- Adicionar propriedades estruturais (invariantes) se necessário
- Considerar screenshot tests se regressões visuais em imagens forem críticas
