<objective>
Add 80-column test cases to all four components and regenerate all golden files to match the fixed spec.

Per `arquitetura-teste.md` §"Matriz de Cobertura", the full coverage matrix requires **3 widths: 30, 60, 80** for all components. Currently only 30 and 60 exist.

**Target file counts after this plan:**
- Messages: 6 kinds × 3 widths × 2 files = **36 files**
- CommandBar: 5 scenarios × 3 widths × 2 files = **30 files**
- DecisionDialog: 10 scenarios × 3 widths × 2 files = **60 files** (NOTE: not 90 — the spec's 45 pairs uses 5 severities × 3 layouts × 3 widths, but our test suite uses 10 representative scenarios × 3 widths)
- HelpModal: 4 variants × 3 widths × 2 files = **24 files** (NOTE: spec says 6 pairs = 12 files, but our suite covers 4 variants: 3actions + 15actions×3-scroll-positions)

**Total: 36 + 30 + 60 + 24 = 150 files** (plus the 80-col variants of no-actions which are 2 each = same count already in 5 scenarios).

Purpose: The spec requires 3 widths for complete coverage. The 80-col case is the "standard" layout — it's the most important width for catching regression in the ideal layout. The `-update` run after adding 80-col cases regenerates ALL golden files simultaneously, ensuring the fixed parser format is embedded in all files.

Output: Updated test files with 80-col cases; all golden files regenerated via `-update` run; tests passing on second run.
</objective>
