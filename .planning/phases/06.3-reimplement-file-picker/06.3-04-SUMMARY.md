### Task 2: Fix filepicker_test.go + add 18 behavioral Update() tests

**Files modified:**
- `internal/tui/filepicker_test.go` (full rewrite)
- `internal/tui/filepicker.go` (bug fix: infinite loop in path truncation)
- `internal/tui/testdata/golden/filepicker-initial-80.txt.golden` (updated)
- `internal/tui/testdata/golden/filepicker-initial-80.json.golden` (updated)

**Changes:**
- Fixed `newTestFilePickerModal()` to set `ext: ".abditum"` and `mode: FilePickerOpen`
- Fixed all tests to call `Init()` before manipulating state (Init resets focusPanel to 0)
- Fixed all tests to set `focusPanel` AFTER `Init()` call
- Replaced all `fpk.loadDirectory()` calls with `fpk.loadFilesForCursor()`
- Removed all references to `fpk.directories` (old field, gone)
- `TestFilePickerModalInit`: asserts `focusPanel == 0` after Init (was asserting non-zero)
- `TestFilePickerModalShortcuts`: asserts exactly 2 shortcuts (Tab Painel + F1 Ajuda)
- `TestFilePickerModalDisplaysFileSizes`: checks for space-separated units (" B", " KB", " MB")
- `TestFilePickerModalDisplaysRelativeDates`: checks for "/" (absolute date format) not "now"/"ago"
- `TestFilePickerModal_Golden`: injects fixed timeFmt and fixed display path for determinism
- Added `TestFilePickerUpdateBehavior` with 18 table-driven behavioral cases from D-07

**Bug fixed in `filepicker.go`:** `renderCaminhoHeader` path truncation used byte-level iteration that caused an infinite loop on Windows paths (backslash + multi-byte UTF-8 `…` character). Replaced with rune-based slice truncation.
