## Implementation Decisions

### D-01: MsgKind — split MsgInfo em dois tipos

`MsgInfo` atual é renomeado para `MsgSuccess`. Um novo `MsgInfo` é adicionado.

```go
const (
    MsgSuccess MsgKind = iota // operação concluída — semantic.success (#9ece6a), TTL 3s default
    MsgInfo                   // informação neutra — semantic.info (#7dcfff), TTL 3s default
    MsgWarn                   // atenção — semantic.warning (#e0af68), permanente, clearOnInput
    MsgError                  // falha — semantic.error (#f7768e) + bold, TTL 5s default
    MsgBusy                   // spinner — accent.primary (#7aa2f7), permanente
    MsgHint                   // dica contextual — text.secondary (#565f89) + italic, permanente
)
```

Spinner frames: `[]string{"◐", "◓", "◒", "◑"}` — Frame 0-3, avança em `Tick()`.

TTL defaults (valores usados quando o chamador não especifica outro):
- `MsgSuccess`: 3 ticks
- `MsgInfo`: 3 ticks
- `MsgBusy`: 0 (permanente — ignora ttlSeconds)
- `MsgWarn`: 0 (permanente, clearOnInput=true por convenção)
- `MsgError`: 5 ticks
- `MsgHint`: 0 (permanente)

### D-02: Action — Priority, HideFromBar e Group como int

```go
type Action struct {
    Keys        []string
    Label       string
    Description string
    Group       int         // agrupamento no Help modal — grupos em ordem crescente
    Scope       ActionScope
    Priority    int         // maior = mais à esquerda na barra; governa truncação
    HideFromBar bool        // oculto da barra de comandos; listado no Help
    Enabled     func() bool
    Handler     func() tea.Cmd
}
```

Impacto da mudança `Group string → int`:
- `root.go`: `Group: "Global"` → `Group: 1`
- `help.go`: `buildContent` precisa de `ActionManager.GroupLabel(group int) string`
- `ActionManager.RegisterGroupLabel(group int, label string)` adicionado

### D-03: RenderCommandBar — spec-correto com âncora F1

**Tokens:**
- Tecla (F2, ctrl+Q, F1): `accent.primary` (#7aa2f7) + bold
- Label (Dica de uso, Sair, Ajuda): `text.primary` (#a9b1d6)
- Separador `·`: `text.secondary` (#565f89)
- 2 espaços de padding à esquerda

**Algoritmo:**
1. Coletar `Visible()` = `Enabled() == true` AND `HideFromBar == false`, ordenadas por Priority decrescente
2. Detectar âncora: action com `Keys[0] == "f1"` é extraída e posicionada à direita
3. Body (demais ações): `"  " + strings.Join(parts, sepStyle.Render(" · "))`
4. Combinar body + anchor usando `lipgloss.NewStyle().Width(width).Align(lipgloss.Right)` para o anchor

**F1 como âncora:**
- F1 é `HideFromBar: false` — aparece na barra como âncora direita
- Enquanto modal está ativo, `Shortcuts()` do modal substitui `RenderCommandBar` — F1 desaparece automaticamente

### D-04: RenderMessageBar — função exportada em messages.go

```go
// RenderMessageBar renderiza a barra de mensagens (1 linha) com borda continuua.
// Quando msg == nil, renderiza borda simples. Exported para uso pelo PoC.
func RenderMessageBar(msg *DisplayMessage, width int) string
```

**Anatomia** (spec-correta):
```
── Texto da mensagem ────────────────────────────────────────────────────────
```

Regras:
- Borda `─` sempre visível na largura total (border.default #414868)
- Quando há mensagem: `"── "` + símbolo+texto + `" ─"` + `─` até a largura
- Símbolo por kind renderizado em `lipgloss.NewStyle().Foreground(token)`
- 2 `─` no início antes do símbolo

Tokens de símbolo:
| Kind | Símbolo | Cor hex | Atributo |

| MsgSuccess | "✔" | #9ece6a | — |
| MsgInfo | "ℹ" | #7dcfff | — |
| MsgWarn | "⚠" | #e0af68 | — |
| MsgError | "✗" | #f7768e | bold |
| MsgBusy | frame do spinner | #7aa2f7 | — |

### D-07: poc-mensagens — estrutura do model

```go
// cmd/poc-mensagens/poc_model.go
type pocMessagesModel struct {
    width   int
    height  int
    actions *tui.ActionManager
    msgs    *tui.MessageManager
}

func newPocModel() *pocMessagesModel { ... }

func (m *pocMessagesModel) Init() tea.Cmd {
    return tea.Tick(time.Second, func(t time.Time) tea.Msg { return tui.TickMsg(t) })
}
```

**tickMsg exportado:** `type TickMsg time.Time` em `internal/tui` — necessário para que o PoC possa enviar o tick para o `MessageManager.Tick()`. Se Claude preferir, pode usar um tipo local no PoC com uma interface adapter — Claude decide pela opção mais limpa sem alterar o tickMsg.

### Claude's Discretion

- Exportar ou não `tickMsg`: se necessário para o PoC, exportar como `TickMsg time.Time`. Se possível manter unexported usando outro mecanismo (ex: função helper exportada), Claude decide
- `clearOnInput` no PoC: pocModel deve chamar `msgs.HandleInput()` em `KeyPressMsg` (segue padrão de rootModel)
- Testes unitários para ajustes: mínimo são os tests de `messages_test.go` passando com os novos tipos; testes do PoC em si são opcionais
- Shift+Fx no Bubbletea v2: verificar padrão correto do projeto (pode ser "shift+f2", "F2" ou outro) — usar o mesmo padrão dos testes existentes
- Padding e layout do header do PoC: `lipgloss.NewStyle().Width(m.width)` com `"  Abditum"` — estilo simples
