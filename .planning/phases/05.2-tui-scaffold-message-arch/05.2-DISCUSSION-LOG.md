## Decisions Captured

### D-01: MsgKind enum split

**Context:** Existing `messages.go` has `MsgInfo MsgKind = iota` which semantically
maps to success (iota=0 was the first success-like message type). DS defines 6 distinct
message semantics.

**Decision:** Rename existing `MsgInfo` → `MsgSuccess` (keeps iota=0). Add new
`MsgInfo` (information neutral, semantic.info) as iota=1. Full enum:

```
MsgSuccess = 0  // ✓ semantic.success
MsgInfo    = 1  // ℹ semantic.info
MsgWarn    = 2  // ⚠ semantic.warning
MsgError   = 3  // ✗ semantic.error (bold)
MsgBusy    = 4  // ◐◓◑◒ accent.primary (spinner)
MsgHint    = 5  // • text.secondary italic
```

### D-01a: Single MsgHint for all hint variants

**Context:** DS defines two hint subtypes — "Dica de campo" (field-level, triggered by
field focus in dialogs) and "Dica de uso" (proactive app hints). Architecture doc only
has `MsgHint`.

**Decision:** Single `MsgHint` covers both semantics. Distinction is in the message
text content only — both render identically (`•` text.secondary italic). Callers choose
which text is appropriate.

### D-02: Action struct additions

**Context:** `actions.go` has `Action.Group string` with no Priority or HideFromBar.
The spec requires grouping by int for help modal ordering and priority for bar truncation.

**Decision:**
- `Group string` → `Group int`
- Add `Priority int` (higher = leftmost in bar; truncation removes lowest first)
- Add `HideFromBar bool` (true = dispatches normally + appears in Help, NOT in bar)
- Add `ActionManager.RegisterGroupLabel(group int, label string)` for help modal section titles

**Semantics clarified:**
- `Enabled() == false` → action NOT in bar AND NOT dispatched (invisible everywhere except Help)
- `HideFromBar == true` → action dispatches normally, appears in Help, NOT in bar

### D-03: RenderCommandBar rewrite

**Context:** Existing `RenderCommandBar` uses ANSI escape codes with wrong colors,
separator `"  |  "` instead of `·`, no F1 right anchor.

**Decision:** Rewrite with:
- Tokens: key=`accent.primary` bold, label=`text.primary`, separator `·`=`text.secondary`
- `Visible()` = `Enabled()==true AND HideFromBar==false`, sorted by Priority desc
- F1 extracted from Visible() and rendered as fixed right anchor
- Available width = total − len("F1 Ajuda") − separators; lower-priority actions dropped on truncation
- Enabled=false actions do NOT appear (not dim — per spec Comportamento section)

**Note:** There is a contradiction in the spec between the wireframe section (shows
disabled action as `text.disabled` dim) and the Comportamento section ("remove a ação
da barra, não fica exibida como dim"). We follow the Comportamento section text as the
authoritative statement.

### D-04: RenderMessageBar exported function

**Context:** No exported render function exists; rootModel renders inline. PoC living in
a different package needs to call this directly.

**Decision:** Export `RenderMessageBar(msg *DisplayMessage, width int) string` from
`messages.go`. Anatomy:
- Always renders 1 line, full width, border.default `─`
- When msg==nil: solid `─────...`
- When msg present: `── ` + colored symbol + ` ` + text + ` ─` + fill `─` to width
- Truncate text with `…` if message exceeds available width

Symbol table (from DS — Ícones e Símbolos + DS — Mensagens):
```
MsgSuccess | ✓  | semantic.success #9ece6a | —
MsgInfo    | ℹ  | semantic.info    #7dcfff | —
MsgWarn    | ⚠  | semantic.warning #e0af68 | —
MsgError   | ✗  | semantic.error   #f7768e | bold
MsgBusy    | ◐◓◑◒ | accent.primary #7aa2f7  | — (1 frame/tick)
MsgHint    | •  | text.secondary   #565f89 | italic
```

### D-07: pocMessagesModel structure

`pocMessagesModel` implements `tea.Model` directly (not via childModel interface).
Fields: `actions *tui.ActionManager`, `messages *tui.MessageManager`, `width`, `height`.
Method `renderFrame()` composes: header + workArea (lipgloss.Place) + messageBar + commandBar.
Init sends `tea.Tick(time.Second, ...)` to drive MessageManager.Tick().

## Visual Spec Extraction

Sections read from `tui-specification-novo.md`:

### Help Modal (lines 602–900)
- Wireframe: `╭── Ajuda — Atalhos e Ações ──...──╮`
- Footer: `╰──...── Esc Fechar ──╯`
- Tokens: title=text.primary bold, group labels=text.secondary bold, keys=accent.primary, descriptions=text.primary, border=border.default
- Scroll: ↑/↓ arrows + ■ thumb on right border, all in text.secondary
- Content: ActionManager.All() grouped by Group int (ascending), actions by Priority desc within group
- "Esc Fechar" always visible at bottom (even when other shortcuts are modal shortcuts)

### Barra de Comandos (lines 901–1000)
- 1 linha fixa, última linha da tela
- Tokens: key=accent.primary bold, label=text.primary, separator `·`=text.secondary, F1=accent.primary bold
- F1 is fixed right anchor; reserved space deducted before distributing other actions
- Enabled=false → action disappears from bar (per Comportamento section — not dim)
- HideFromBar=true → action dispatches, in Help, NOT in bar
- Priority governs truncation (lowest priority dropped first)
- During dialog: bar shows dialog's internal actions only (ActionManager actions hidden)

### Barra de Mensagens (lines 1001–1100)
- 1 linha fixa between work area and command bar
- Border `─` always visible (border.default), functions as separator
- Text embedded in border: `── ✓ Gmail copiado... ───────────`
- Symbol+text starts with 2-space left padding (aligned with header text)
- Truncate with `…` when message exceeds available width
- Border color stays border.default regardless of message type — only embedded text uses semantic color
- One message at a time; new message replaces immediately
- TTL expiry → returns to bare `─` border
