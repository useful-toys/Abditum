**Gathered:** 2026-03-31
**Status:** Ready for planning

<decisions>
## Implementation Decisions

### Versioning e Deserialização

**D-01: `DeserializarCofre(data []byte, version uint8)` — versão como parâmetro**
- Assinatura muda de `DeserializarCofre(data []byte)` para `DeserializarCofre(data []byte, version uint8)`
- A versão já está disponível no header do arquivo (byte 4); o `storage` passa o valor ao chamar
- Internamente, compat fields nas structs `*JSON` tratam renomeações entre versões:
  ```go
  type segredoJSON struct {
      Nome    string `json:"nome"`    // v1+
      NomeLeg string `json:"name"`   // v0 legado — exemplo de compat field futuro
  }
  // no conversor: if version == 0 && raw.Nome == "" { nome = raw.NomeLeg }
  ```
- Regra: compat fields para renomeações e defaults; structs por versão apenas se estrutura mudar de nível hierárquico

**D-02: `migrate.go` removido**
- O arquivo `internal/storage/migrate.go` e seu tipo `MigrationFunc` são deletados
- A lógica de compatibilidade retroativa pertence ao deserializador, não a transformações JSON→JSON
- Testes relacionados ao `migrate.go` também são removidos ou adaptados

### Encapsulamento — Métodos Privados nas Entidades

**D-03: `segredo.marcarModificacao()` — entidade gerencia seu próprio timestamp**
- Método privado em `Segredo`:
  ```go
  func (s *Segredo) marcarModificacao() {
      s.dataUltimaModificacao = time.Now().UTC()
  }
  ```
- Manager chama após mutação de conteúdo confirmada; elimina acesso direto a `segredo.dataUltimaModificacao` espalhado em múltiplos métodos

**D-04: `cofre.marcarModificado()` — Cofre gerencia seu próprio estado**
- Método privado em `Cofre`:
  ```go
  func (c *Cofre) marcarModificado() {
      c.modificado = true
      c.dataUltimaModificacao = time.Now().UTC()
  }
  ```
- Manager chama após qualquer mutação real; elimina acesso direto a campos internos do Cofre

**D-05: Factory define `estadoSessao` inicial**
- `pasta.criarSegredo()` retorna segredo já com `estadoSessao = EstadoIncluido` (estado correto para criação)
- `pasta.duplicarSegredo()` retorna segredo já com `estadoSessao = EstadoIncluido`
- Manager não seta `estadoSessao` após chamar o factory — regra de estado pertence à entidade
- Atual: Manager faz `segredo.estadoSessao = EstadoModificado` após o factory — isso é corrigido para `EstadoIncluido`

**D-06: Deep copy via métodos privados das entidades**
- `Pasta.copiarProfundo(filtrarExcluidos bool) *Pasta` — cópia da subárvore com referências populadas
- `Segredo.copiar() *Segredo` — cópia profunda com `[]byte` independentes
- `CampoSegredo.copiar() CampoSegredo` — cópia com valor zerado-safe
- Manager.`prepararSnapshot()` delega para `cofre.pastaGeral.copiarProfundo(true)` — deixa de acessar campos internos diretamente
- `copiarModelo()` permanece no Manager (modelos não têm lógica de identidade relevante; cópia é operação trivial de persistência)

**D-07: `segredo.zerarValoresSensiveis()` — zeragem encapsulada na entidade**
- Método privado em `Segredo`:
  ```go
  func (s *Segredo) zerarValoresSensiveis() {
      for i := range s.campos {
          if s.campos[i].tipo == TipoCampoSensivel {
              crypto.Wipe(s.campos[i].valor)
              s.campos[i].valor = nil
          }
      }
      crypto.Wipe(s.observacao.valor)
      s.observacao.valor = nil
  }
  ```
- Manager.`limparCamposSensiveis()` torna-se loop simples: `for _, s := range pasta.segredos { s.zerarValoresSensiveis() }`
- Entidade conhece quais campos são sensíveis e como zerá-los

### Correção de Bug

**D-08: `AlternarFavoritoSegredo` NÃO atualiza `segredo.dataUltimaModificacao`**
- Favoritar é preferência de navegação, não edição de conteúdo (per `arquitetura-dominio.md` §7)
- Bug atual: Manager chama `segredo.dataUltimaModificacao = now` em `AlternarFavoritoSegredo`
- Correção: remover essa linha; apenas `cofre.marcarModificado()` é chamado
- `segredo.marcarModificacao()` NÃO é chamado em favoritar

### Testes

**D-09: Teste explícito para semântica de favoritar**
- Adicionar teste em `manager_test.go` que verifica: após `AlternarFavoritoSegredo`, `segredo.DataUltimaModificacao()` não muda
- Cobre o bug corrigido em D-08 contra regressão futura

<code_context>
## Existing Code Insights

### Reusable Assets
- `crypto.Wipe()` — já usado em `limparCamposSensiveis`; será chamado a partir do método privado da entidade
- Structs `*JSON` em `serialization.go` — já são privadas ao arquivo; podem receber compat fields sem impacto externo

### Established Patterns
- Padrão validar/mutar já estabelecido nas entidades: `validarX()` seguido de `x()` retornando `(bool, error)`
- `time.Now().UTC()` é o padrão de timestamp usado em todo o Manager

### Integration Points
- `internal/storage/storage.go` chama `vault.DeserializarCofre(data)` — precisará passar `version` (já disponível no ponto de chamada como `header.Version`)
- Todos os 5 métodos do Manager que fazem `segredo.dataUltimaModificacao = now` serão simplificados
- `Manager.prepararSnapshot()` delega deep copy para entidade; `Manager.Salvar()` não muda interface

</code_context>

<specifics>
## Specific Ideas

- O estado inicial correto para segredo recém-criado é `EstadoIncluido` (não `EstadoModificado` como está hoje) — segredo que nunca existiu no arquivo não é "modificado", é "incluído"
- Compat fields devem ter comentário inline indicando a versão de origem: `// v0 legado`
- `copiarProfundo` em `Pasta` precisa repovoar referências pai-filho (`subCopia.pai = copia`, `segredoCopia.pasta = copia`) — mesma responsabilidade que `popularReferencias` na deserialização

</specifics>
